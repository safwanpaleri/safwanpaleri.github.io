Unity LLM Evalution Tool

Description

As part of my MSc dissertation, I designed and developed a Unity-based tool to integrate and evaluate multiple Large Language Models (LLMs) for in-game dialogue generation. The project aimed to bridge a key gap in game development by providing developers with a native Unity framework to test, compare, and refine LLM-driven NPC interactions. Beyond technical implementation, the work explored both the strengths and limitations of LLMs in game contexts, contributing insights to future game AI integration.

Implementations

  • Multi-Agent Conversation System: Built a roundtable-style multi-NPC interaction system where each agent was powered by a different LLM, supporting turn-taking, interruptions, and dynamic dialogue flow.
  • Evaluation Framework: Designed metrics-based evaluation (Coherence, Relevance, Naturalness, Engagement, Contextual Accuracy, and Response Time) inspired by recent literature, with automated scoring via a secondary LLM.
  • Automated Testing Scene: Implemented stress-testing mode with up to 100 test cases per LLM, logging results into JSON, and visualizing performance via graphs and tables.
  • Interactive Artifact Scene: Enabled real-time NPC conversations with player participation (voice-to-text input), showcasing immersive applications of LLMs in games.
  • Visualization & Feedback UI: Created dynamic Unity UI panels for live metric display, past score tracking, and comparison across multiple LLMs.
  • API Integration Layer: Developed extensible Unity scripts to integrate different LLMs (ChatGPT, Gemini, Claude, Mistral, DeepSeek), ensuring future scalability.
  • Performance Analysis: Conducted comparative benchmarking across models, highlighting strengths (e.g., ChatGPT’s fluency, Gemini’s structured output) and weaknesses (e.g., rate limits in Mistral, latency in DeepSeek).

Procedural Generation

Description

Developed a 2.5D procedurally generated maze ball game using C++ and DirectX. The project focuses on gameplay mechanics, procedural content generation, collision detection, and real-time post-processing effects, offering an engaging and scalable game experience. This is created as part of the university module

Implementations

  • Procedural maze generation using Randomized Depth First Search (RDFS).
  • Responsive ball control and AABB-based collision detection system.
  • Dynamic camera system for 2.5D gameplay view.
  • Real-time post-processing effects with GPU shaders (e.g., glow, lava pulse, god rays).
  • UI customization using ImGui to adjust gameplay parameters (ball speed, maze size, effects).
  • Scalable difficulty with increasing maze size each level.
